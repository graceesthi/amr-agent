# data/

## What is in here

| Path | Contents | Generated by |
|------|----------|--------------|
| `seed/*.md` | The document corpus that `search_amr_literature` searches | `scripts/build_dataset.py` |
| `surveillance.json` | Structured records behind `get_resistance_profile` and `compare_regions` | `scripts/build_dataset.py` |
| `adversarial/poisoned_bulletin.md` | A deliberately poisoned document used by `tests/test_security.py` | `scripts/build_dataset.py` |

Populate everything with:

```bash
python scripts/build_dataset.py
```

---

## ⚠️ The shipped data is synthetic

Every figure in `seed/` and `surveillance.json` is **fabricated**. The series are
internally consistent, epidemiologically plausible in shape, and consistent
between the narrative documents and the structured table — but they are not real
surveillance data and must not be cited as such. Each generated file carries a
banner saying so.

They exist for one reason: the repository-quality gate requires that
`python src/agent.py` works from a clean clone, and that is not achievable if the
first step is "download and clean 200 MB of surveillance reports."

## Using real data

The retrieval pipeline is source-agnostic. It indexes every `.md` and `.txt`
file in `seed/` (recursively, `README*` excluded). To switch to real sources:

### 1. Documents

Drop the reports in as `.md` or `.txt`. Useful public sources:

| Source | What it gives you | Where |
|--------|-------------------|-------|
| WHO GLASS reports | Global AMR surveillance, country-level | who.int → GLASS |
| ECDC EARS-Net / Surveillance Atlas | European resistance by country, organism, year | ecdc.europa.eu |
| CDC NARMS / AR Threats Report | US resistance, foodborne and clinical | cdc.gov |
| ECDC/EFSA JIACRA | Consumption linked to resistance | ecdc.europa.eu |
| PubMed / Europe PMC open access | Mechanism and intervention literature | europepmc.org |

Practical notes:

- **Convert PDFs to text first.** The pipeline does not parse PDF. `pdftotext
  -layout` preserves table structure better than the default mode, which matters
  because most of the numbers you want are in tables.
- **Keep one document per report.** Parent-child chunking works from headings;
  concatenating five reports into one file produces parents that span topics and
  measurably degrades `context_precision`.
- **Preserve headings.** The first `#` heading becomes the passage title shown
  to the model and used in citations.
- **Strip page furniture** — running headers, footers, page numbers. They chunk
  into children that match nothing and dilute the BM25 statistics.

### 2. Structured records

Replace `surveillance.json`, keeping the shape:

```json
{
  "records": [
    {
      "organism": "Klebsiella pneumoniae",
      "region": "Southern Europe",
      "year": 2023,
      "antibiotic_class": "carbapenems",
      "resistance_pct": 35.0,
      "isolates_tested": 1204,
      "source": "ECDC EARS-Net 2023"
    }
  ]
}
```

Field notes:

- `organism` — full binomial. Clinical shorthand ("MRSA", "K. pneumoniae") is
  resolved by the alias table in `src/surveillance.py`; extend `ALIASES` there
  if your sources use names it does not know.
- `region` — must be spelled consistently across all records. `compare_regions`
  matches on the exact string (case-insensitively, with unique-substring
  fallback).
- `resistance_pct` — percentage, not a fraction. `35.0`, not `0.35`.
- `isolates_tested` — the denominator. Do not omit it; the agent uses it to
  caveat small-sample series, and without it the caveat silently disappears.
- `source` — real attribution. This string is what a reader traces back.

### 3. Regenerate the evaluation set

The question set and its reference answers are derived from the data:

```bash
python eval/build_questions.py
```

Skipping this after a data change leaves references pointing at figures that no
longer exist, and `context_recall` collapses for reasons that have nothing to do
with the retriever.

### 4. Re-run the evaluation

```bash
python eval/run_ragas.py --arm both --report
```

RAGAS numbers do not transfer across corpora. A new corpus needs a new baseline.

## Licensing

Check the licence of anything you add here before committing it. WHO and ECDC
publications are generally reusable with attribution; journal articles usually
are not. `data/seed/` is in version control, so a licence mistake here is a
licence mistake in your git history.
